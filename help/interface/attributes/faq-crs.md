# Questions fréquentes, restrictions et bonnes pratiques

Questions fréquentes et bonnes pratiques sur les attributs du client dans Analytics et Target.

## Bonnes pratiques et restrictions  {#section_7F5189B3DAA84EE6865B91D2026EE05A}

Conseils et limites lors de l’utilisation des attributs client.

| Problème | Description |
|--- |--- |
| [!UICONTROL Limites des abonnements d’attributs] du client | Lorsque vous procédez à la mise à niveau vers Analytics Premium, un délai de 24 heures est nécessaire avant que des attributs supplémentaires soient disponibles. Il se peut que l’erreur Nombre maximal d’abonnements des attributs s’affiche durant ce délai. |
| Connexions multiples sur le même périphérique | Lors de l’utilisation d’attributs du client pour télécharger des profils client dans une source de données, Adobe recommande de ne pas autoriser les utilisateurs qui partagent le même périphérique (c’est-à-dire le même ID Experience Cloud). Ce faisant, le service ECID, qui persiste sur le périphérique, peut lier plusieurs utilisateurs sous le même ID Experience Cloud, ce qui entraîne des résultats inattendus [!DNL Target]. **Remarque :** Pour Mobile, l’ECID est permanent après l’installation de l’application Mobile et vous devez réinstaller l’application pour générer un nouvel ECID. Pour le Web, un nouvel ECID est généré une fois le cookie du navigateur effacé. |
| Limite de transfert de fréquence quotidienne | Adobe recommande de ne mettre à jour les attributs du client qu’une seule fois par jour. Vous devez attendre au moins 24 heures pour télécharger un autre fichier de données de profil client pour le même ensemble de profils. |
| Identifiant Analytics personnalisé (`s.visitorID`) | La définition d’un ID de client à l’aide de `s.visitorID` permet d’identifier les utilisateurs dans Analytics. Cependant, les intégrations dans lesquelles les données Analytics sont exportées ou importées à l’aide du service d’ID ne fonctionneront pas lorsqu’un visiteur est identifié à l’aide de `s.visitorID.`<br>Ceci inclut, sans s’y limiter, les audiences partagées, Analytics pour Adobe Cible (A4T) et les attributs du client.<br>Pour ces intégrations, la définition d’un Analytics ID personnalisé n’est pas prise en charge. |
| Limites de longueur des caractères dans Analytics | Lors de la création d’un abonnement Analytics, la longueur des champs des fichiers téléchargés est tronquée à 255. |

## Questions fréquences sur les attributs du client  {#section_E47866EEA83348E09FE43CEC5E44C461}

| Question | Réponse |
|--- |--- |
| Puis-je recevoir des notifications sur l’état de transfert des attributs du client ? | Oui. Voir [Gestion des notifications](https://docs.adobe.com/content/help/en/core-services/interface/manage-users-and-products/organizations.html#concept_0105453AD71847B8BFCAF4A40915F157). |
| Que dois-je faire pour commencer à utiliser les attributs du client ? | <ol><li>Recevez les privilèges d’accès. Si vous êtes un client d’Analytics, Adobe met en service les attributs client pour vous. Si vous utilisez uniquement Adobe Cible et ne possédez pas Analytics, vous devez demander la mise en service des services principaux en contactant le service d’assistance clientèle.</li> <li>Contactez votre équipe CRM. Découvrez quels types de données clients sont disponibles et qui pourraient être utiles dans Analytics et dans Experience Cloud.</li><li>Mise en oeuvre des services principaux. Voir [Activation de vos solutions pour les services](https://docs.adobe.com/content/help/fr-FR/core-services/interface/about-core-services/core-services.html) principaux pour connaître les étapes de modernisation de votre mise en oeuvre. (Voir la section sur la synchronisation des identifiants de client pour obtenir des informations importantes.)</li></ol> **Remarque :** Un FAQ destiné aux administrateurs pour la mise en œuvre de services principaux est disponible  [ ici](https://docs.adobe.com/content/help/en/core-services/interface/manage-users-and-products/faq.html#concept_13219B4E51784577B6FF78AAA203DE91). |
| Combien d’attributs de client suis-je autorisé à utiliser ? | You can upload hundreds of `.csv` columns to the customer attribute service. Cependant, lors de la configuration des abonnements et de la sélection des attributs, les limites suivantes s’appliquent (par suite de rapports), en fonction des solutions que vous possédez :  <ul><li>Foundation : 0</li><li>Select : 3</li><li>Prime : 15</li><li>Ultimate : 200</li><li>Standard : 3 au total</li><li>Premium : 200</li><li>Adobe Cible Standard : 5</li><li>Adobe Cible Premium : 200</li></ul> |
| La migration vers le service d’identification d’Experience Cloud est-elle requise ? | La migration dépend des solutions que vous utilisez. <ul><li>Adobe Analytics :  Fortement recommandé </li><li>Adobe Cible : Obligatoire. </li></ul><br> L’utilisation du service d’ID permet d’améliorer les fonctionnalités et d’utiliser les dernières fonctionnalités d’Experience Cloud, notamment les audiences en temps réel, la modernisation des Cibles Adobe, l’intégration d’Analytics et le suivi de pulsation vidéo. <br> Pour plus d’informations, voir [Activation de vos solutions pour les services](https://docs.adobe.com/content/help/fr-FR/core-services/interface/about-core-services/core-services.html)principaux. <br>** Remarque :** Le service [d’ID](https://docs.adobe.com/content/help/fr-FR/id-service/using/intro/overview.html) Experience Cloud est l’implémentation modernisée de ce qui était auparavant appelé service d’ID de visiteur _Analytics._ |
| Comment la fonctionnalité d’attributs du client se rapporte-t-elle à Adobe Audience Manager ? | Audience Manager peut recevoir des données pour identifier l’audience, mais il ne peut pas exécuter des fonctions d’analyse qui lient les attributs aux données comportementales historiques ou fournir des fonctions de création de rapports, d’analyse et de segmentation disponibles dans Adobe Analytics. [!UICONTROL Les personnes] permettent que les données enrichies issues de différentes solutions soient associées ensemble et associées à un identifiant unique à utiliser dans Experience Cloud. <br>Dans Adobe Cible, les attributs du client apparaissent sous la forme d’attributs individuels pouvant être combinés à d’autres règles pour créer des audiences. Les Audiences partagées avec le service [!UICONTROL Personnes] sont des audiences complètes qui ne peuvent pas être modifiées. |
| **(Analytics uniquement)** En quoi cette fonctionnalité diffère-t-elle de celle fournie dans Analytics Premium ? | Par le passé, les clients qui souhaitaient combiner les données d’attributs du client avec les données Analytics se fiaient principalement à l’Data Workbench associé à cette fonction. Les attributs du client exposent cette fonction à une audience plus large en fournissant les attributs du client sous forme de dimensions et de mesures dans les Reports &amp; Analytics, les Ad Hoc Analysis et le Report Builder. Les clients d’Analytics Standard auront accès aux attributs du client, mais avec des fonctionnalités limitées. La fonctionnalité complète est disponible pour les clients Analytics Premium. |
| **(Adobe Cible uniquement)** Puis-je précharger ou télécharger des données pour les clients qu’Adobe Cible n’a jamais vus ? | Oui. Lorsque le visiteur envoie sa première requête à Adobe Cible, le système récupère les informations existantes que nous possédons à leur sujet dans Attributs du client et les utilise pour le ciblage. **Remarque :**  La récupération de ces données peut prendre jusqu’à 20 minutes après la première interaction du visiteur avec Adobe Cible. |
| **(Adobe Cible uniquement)** Puis-je créer une super audience en combinant les données d’attributs du client avec les données d’audience partagées ? | Non. Les données d’audience partagée sont une audience terminée. |
| **(Adobe Cible uniquement)** Comment la fonctionnalité des attributs du client se compare-t-elle à l’API de profil en vrac d’Adobe Cible ? | L’API de profil en masse permet aux profils de Cible Adobe d’être mis à jour directement via l’API, soit pour un profil individuel, soit en bloc. La fonctionnalité est similaire aux attributs du client, avec les différences clés suivantes :<ul><li>L’API de profil est un appel d’API REST et les attributs du client utilisent FTP.</li><li>L’API profil d’Adobe Cible envoie uniquement des données à Adobe Cible plutôt qu’à l’ensemble d’Experience Cloud.</li><li>Les attributs du client offrent une interface simple pour créer et gérer ces données externes.</li></ul> |
| **(Adobe Cible uniquement)** Le transfert de données à partir des attributs du client vers Adobe Cible s’étend-il à la durée de vie du profil des visiteurs Cibles d’Adobe ? | Oui. Voir [Durée de vie du profil du visiteur](https://docs.adobe.com/content/help/en/target/using/audiences/visitor-profiles/visitor-profile.html) dans l’aide d’Adobe Target. |
| **(Adobe Cible uniquement)** Puis-je cible sur les données transférées dans les attributs du client immédiatement après l’identification du visiteur par l’identifiant du client ? | Oui. Lors de l’appel du serveur à Adobe Cible, qui inclut l’identifiant tiers de la mbox, toutes les données d’attributs du client seront disponibles. |
| **(Adobe Cible uniquement)** Que représente la colonne État **[!UICONTROL de la]** synchronisation pour les fichiers téléchargés dans la source d’attributs du client ? | Le nombre d’enregistrements publiés et synchronisés par Adobe Cible peut être affiché en cliquant sur l’icône Synchroniser l’état en regard d’un fichier d’attributs spécifique. `Sync %` est une mesure en temps réel qui spécifie le pourcentage de profils synchronisés dans Adobe Cible.<br> **Remarque :** La synchronisation des attributs avec Adobe Cible peut prendre jusqu’à 24 heures. |
| Que représentent les mesures de transfert de fichiers dans la source des attributs du client ? | Vous pouvez vérifier l’état des attributs chargés dans les attributs du client à l’aide des mesures suivantes : <ul><li>Enregistrements :  Nombre d&#39;enregistrements dans le fichier d&#39;attributs.</li><li>**Nouveaux enregistrements :** Nombre de nouveaux enregistrements présents dans le fichier d&#39;attributs.</li> <li>**Enregistrements mis à jour :** Nombre d’enregistrements dans qui existent déjà dans les attributs du client avec des valeurs mises à jour dans le fichier.</li><li>**Toutes les données (enregistrements) :** Nombre total d’enregistrements transférés vers les attributs du client avec succès.</li></ul> |
